# v1.1.0

**telvm v1.1.0** is a major refresh of the **companion** operator dashboard (Phoenix LiveView). The UI is organized around three primary tabs—**Pre-flight** (`/health`), **Warm assets** (`/warm`), and **Machines** (`/machines`)—with a shared console shell width and navigation so switching contexts feels like one tool rather than separate pages.

The **mission / lab** area keeps a **preview** column visible with a **fixed-height** preview frame, tighter headings, and the same shell constraints as Warm and Pre-flight so layout stays predictable on typical laptop widths. **Light** and **dark** modes plus **accent** choices (for example purple, yellow, green) help operators match preference or environment.

The **lab image catalog** still offers five **Docker Hub** quick-start images, now as **text labels with Heroicons** only—**Node + Bun**, **Go**, **Elixir + mix**, **python + uv**, and **C + gcc**—without separate static logo files under the app.

**Operator-visible highlights**

- Split **Pre-flight**, **Warm assets**, and **Machines** with consistent shell chrome and routing.
- Improved **preview + lab** column layout and header density on **Machines**.
- **Theme**: light/dark and accent selection for the shell.
- **Catalog**: five hub-backed runtimes; icon + label chips; removal of catalog SVG logos.

**API / upgrade notes**

- **No breaking changes** to the **Machine API** at `/telvm/api` for this release. Agents and scripts using [`agent-api.md`](../agent-api.md) should behave as on v0.1.0 unless you rely on undocumented HTML-only details.
